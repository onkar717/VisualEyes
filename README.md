**VisualEyes** combines real-time metrics collection, a Bubbletea TUI dashboard, AI Root Cause Analysis, and a full React web UI вҖ” giving on-call engineers complete visibility from a single tool.

---

## рҹҢҹ Two Ways to Use VisualEyes

### 1. veye CLI (Terminal-First)
A Bubbletea-powered interactive TUI. Run `veye watch` on your machine and get live metrics, active alerts, pod logs, and AI RCA without opening a browser. Perfect for on-call engineers.

### 2. Hub + Agents (Continuous Monitoring)
Deploy agents to your systems and Kubernetes clusters. Agents push metrics to a central backend with a React dashboard, WebSocket streaming, and PostgreSQL-backed incident history.

---

## вңЁ Features

- рҹ“Ҡ **System metrics** вҖ” CPU, memory, disk, network, load average via `gopsutil`
- вҳёпёҸ **Kubernetes metrics** вҖ” pod-level and node-level stats via kubelet summary API
- рҹ§  **AI-powered RCA** вҖ” Claude AI diagnoses incidents and proposes safe remediation commands
- рҹҡЁ **Alert engine** вҖ” configurable rules with dedup, noise filtering, auto-RCA trigger
- вҡЎ **WebSocket streaming** вҖ” live metric push to React dashboard, no polling
- рҹ“Ҳ **Prometheus `/metrics`** вҖ” plug into any Grafana/Prometheus stack
- рҹ–ҘпёҸ **veye CLI** вҖ” Bubbletea TUI: `status`, `alerts`, `logs`, `rca`, `watch`
- рҹ—„пёҸ **PostgreSQL storage** вҖ” persistent incident history with MTTR tracking
- рҹ”’ **Safety-first RCA** вҖ” remediation commands validated before execution
- рҹҗі **Docker & Kubernetes** вҖ” full containerized deployment, Compose + manifests

## рҹӣ пёҸ Getting Started (Build from Source)

### Prerequisites

- Go 1.24+
- Node.js 18+
- Docker & Docker Compose
- PostgreSQL 14+ (or use Docker Compose вҖ” no local install needed)
- `kubectl` + cluster (for Kubernetes mode)

### 1. Clone & Build

```bash
git clone https://github.com/onkar717/VisualEyes.git
cd VisualEyes

make deps        # Download Go dependencies
make build       # Build server, agents, and veye CLI вҶ’ bin/
make install-ui  # Install frontend dependencies
```

### 2. Configure

```bash
cp .env.example .env
# Set ANTHROPIC_API_KEY for AI RCA (optional вҖ” RCA disabled if not set)
# Set DATABASE_URL for PostgreSQL (optional вҖ” falls back to in-memory)
```

### 3. Run Locally

```bash
./bin/visual-eyes-server   # Backend API on :8080
./bin/visual-eyes-agent    # System metrics push (separate terminal)
make run-ui                # React UI on :5173
open http://localhost:5173
```

### 4. Or вҖ” Full Stack with Docker Compose

```bash
docker-compose up --build -d
# Backend :8080 В· UI :3000 В· PostgreSQL :5432 В· system-agent
```

## рҹ“Ӣ Alert Categories

VisualEyes detects and automatically triggers RCA for:

- **Pod Lifecycle** вҖ” `CrashLoopBackOff`, `OOMKilled`, `ImagePullBackOff`, `Pending`, `CreateContainerConfigError`
- **Resource Pressure** вҖ” CPU throttling, memory saturation, disk pressure, node not-ready
- **Kubernetes Health** вҖ” pod restarts exceeding threshold, deployment replica mismatch
- **Network** вҖ” service endpoint unavailability, DNS resolution failures
- **Storage** вҖ” unbound PVCs, volume mount failures
- **Custom Rules** вҖ” define your own threshold-based alert rules in `configs/config.yaml`
