# Project Documentation

This directory contains the main project documentation for the entire application. The documentation is organized as follows:

## Documentation Structure

- `/docs` (this directory)
  - Project-wide documentation
  - System architecture
  - Integration guides
  - Deployment guides
  - API documentation
  
- `/ui/docs`
  - Frontend-specific documentation
  - Component documentation
  - State management
  - UI/UX guidelines
  
- Backend documentation (in respective folders)
  - API documentation in `/api`
  - Package documentation in `/pkg`
  - Internal documentation in `/internal`

## Getting Started

For new developers, follow these steps to get started with the project:

1. First, read the project overview in `/docs/overview.md`
2. Set up the backend by following `/docs/backend-setup.md`
3. Set up the frontend by following `/ui/docs/setup.md`
4. Read the integration guide in `/docs/integration.md`

## Contributing
